### 1ï¸âƒ£ What is **@HostListener**?

* `@HostListener` is an Angular **decorator**
* It **listens to DOM events** on the **host element**
* When the event occurs, it **executes an event handler method**

ğŸ“Œ **Host element** = the HTML element where the directive is applied

---

### 2ï¸âƒ£ Why use **@HostListener**?

* To react to user actions like:

  * `mouseenter`
  * `mouseout`
  * `click`
  * `keydown`
* Keeps event logic **inside the directive**
* Avoids adding event bindings directly in templates
